## Test Case 7: Welcome SMS sent after CSV enrollment when MDN meets OWL/BISCUIT conditions (AU)

**Test Case ID:** SMS-CSV-WELCOME-001  
**Test Case Title:** Verify Welcome SMS is sent after CSV enrollment for new MDN (CSV registration, brand AU)  
**Priority:** High  
**Automation ID:** C00553  
**Grade:** D  
**Preconditions:**
- OWL/BISCUIT conditions are met: **service enabled: true**, **brand: AU**.
- CSV registration is used for enrollment; service enabled: true, brand: AU for the contract/MDN.
- Postman (or equivalent) is available; access to AWS (SQA/UAT), Titan DB and ISS DB for verification queries.

**Test Steps:**

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | In Postman, prepare an MDN that meets the OWL/BISCUIT conditions in advance (see Test Data for MDN requirements). | MDN/account is ready with ① "使い方サポート" contract, ② Date of birth, ③ gender, ④ brand. |
| 2 | Send the GET request; search for **"smsconfig"** and scroll down. Verify **"isEnabled": true** and **"delayInMins"**: 0, 10, and 20 as per user expectation. | smsconfig shows isEnabled: true and delayInMins (0, 10, 20) as expected. |
| 3 | If **"delayInMins"** is not as per user expectation, set the expected delay time in the PUT request and execute the request. | Delay configuration is updated as required. |
| 4 | Create a CSV file containing the contract(s)/MDN(s) in the format required by the system for enrollment. | CSV file is created with the correct format. |
| 5 | Log in to AWS for the target environment (SQA or UAT). Navigate to the S3 bucket used for enrollment. | Logged in to AWS; correct environment and bucket are available. |
| 6 | Upload the CSV file to the configured S3 bucket path used for enrollment (under **"New Enrollment"** section). | CSV file is uploaded successfully. *(Note: CSV upload runs every 10 minutes.)* |
| 7 | In Titan DB, run: `select * from carrier_integration.carrier_enrollment order by created_datetime desc;` Check that the file entry for the upload appears. | File entry is present in Titan DB; upload is recorded. |
| 8 | In ISS DB, run: `select * from sms_system.message order by created_datetime desc limit 100;` Check the **Welcome** SMS delivery status for the test MDN. | Welcome SMS is scheduled/delivered; status is visible in the results. |

**Expected result (overall):** Welcome SMS will be sent.

**Test Data:**

- **MDN creation requirements (when creating an MDN):**
  - ① "使い方サポート" (usage support) contract  
  - ② Date of birth  
  - ③ Gender  
  - ④ Brand  

- **AWS / environment:**
  - AWS login: https://cloudportal.asurion.com/aws  
  - UAT: https://ap-southeast-2.console.aws.amazon.com/console/home?region=ap-southeast-2#  
  - SQA (S3): https://ap-northeast-1.console.aws.amazon.com/s3/buckets/sj-acyan-sqa-enrollment?region=ap-northeast-1&bucketType=general&tab=objects  

- **Verification queries (備考):**
  - Titan DB (uploaded file entry):  
    `select * from carrier_integration.carrier_enrollment order by created_datetime desc;`
  - ISS DB (scheduled Welcome SMS status):  
    `select * from sms_system.message order by created_datetime desc limit 100;`

**Postconditions:** Test MDN has Welcome SMS sent (or scheduled) as verified in ISS DB. DB rows can be used for traceability. No mandatory cleanup unless test data must be removed per project rules.

**Note:** CSV file is processed every 10 minutes; allow for this when checking Titan DB and Welcome SMS.

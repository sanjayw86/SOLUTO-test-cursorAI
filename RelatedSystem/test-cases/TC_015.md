## Test Case 15: Call Cancellation SMS is sent when reservation is cancelled (Un-provisioned MDN, AU)

**Test Case ID:** SMS-CALL-CANCELLATION-AU-001  
**Test Case Title:** Verify Call Cancellation SMS is sent and recorded in DB when a reservation is cancelled for an un-provisioned MDN (service enabled: true, brand AU)  
**Priority:** High  
**Automation ID:** C00559  
**Grade:** D  
**Preconditions:**
- Call reservation SMS conditions: **MDN un-provisioned**, **service enabled: true**, **brand: AU**.
- A call reservation has been made for the test MDN (via CC tool or Shop UI tool).
- Access to CC tool / Shop UI tool and ISS DB for verification.

**Test Steps:**

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Have one or more un-provisioned MDNs. If needed, enroll via CSV upload or Orange API (un-provisioned, service enabled: true, brand AU). | Un-provisioned MDN(s) are ready. |
| 2 | Go to **CC tool** or **Shop UI tool** and make a reservation using the test MDN. | Reservation is created. |
| 3 | In **CC tool** or **Shop UI tool**, search with the same MDN and **cancel** the reservation. | Reservation is cancelled successfully. |
| 4 | In ISS DB, run: `select * from sms_system.message order by created_datetime desc limit 10;` Identify the **Cancellation SMS** record (Source = SUPPORT_SESSION) for the test MDN. | Call Cancellation SMS record is present in ISS DB. |
| 5 | Verify the message text for the Cancellation SMS in the DB. | Text matches: **【使い方サポート】　お電話のご予約がキャンセルされました。再予約をご希望の場合は、使い方サポートアプリにてご登録いただけます。（このメッセージはご不在等により自動キャンセルされた場合にも送信されます。）専用アプリはこちらからダウンロードしてください。 http://kddi-l.jp/eSd** (Cancellation SMS transmission completed; confirmation via log.) |

**Expected result (overall):** Call Cancellation SMS is sent when the reservation is cancelled (or auto-cancelled) and verifiable in ISS DB.

**Test Data:**

- **MDN creation requirements (when creating an MDN):**
  - ① "使い方サポート" (usage support) contract  
  - ② Date of birth  
  - ③ Gender  
  - ④ Brand  

- **CC Tool:** https://callcenter.soluto-kddi-sqa.jpnmob-acyan.npr.aws.asurion.net/reservation/search  
- **Shop UI Tool:** https://shop.soluto-kddi-sqa.jp.nonprod-asurion53.com/auShop-entry  

- **Verification queries (備考):**
  - ISS DB (Cancellation SMS – Source = SUPPORT_SESSION):  
    `select * from sms_system.message order by created_datetime desc limit 10;`  

**Note:** This message is also sent when the reservation is automatically cancelled (e.g. user absent). Confirmation is via DB/log.

**Postconditions:** Cancellation SMS record exists in ISS DB for the test MDN. No mandatory cleanup unless required by project rules.

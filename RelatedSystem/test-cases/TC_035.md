## Test Case 35: ARCH KFS chat message snack bar on My page

**Test Case ID:** ARCH-CHAT-KFS-NOTIFICATION-SNACKBAR-001  
**Test Case Title:** Verify message snack bar is shown on My page when ARCH sends **KFS** chat message

**Automation ID:** C00632  
**Grade:** D

**Role:** **Operator** (send-side from ARCH).

- **App state:** App is launched.
- **Notification settings:**
  - **ON:** App in-app notification (confirm label in product settings).
  - **ON:** OS push notification.
- **ARCH context:** **KFS** chat flow available for target user; operator can send to the user’s **KFS** session.
- **Skill selection (ARCH):** Select **"[故] Chat (KFS)"** for this test case. Use **"[つか] Message (Soluto)"** only if project scope explicitly requires Soluto validation.
- **Browser:** **Chrome** for ARCH.
- **Environment alignment:** Use one line end-to-end (app + ARCH). Do not mix SQA app with UAT ARCH.

**Test Steps:**

### Phase A — App ready on My page

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Launch app and open **My page**. | My page is displayed and app remains active in foreground. |
| 2 | Stay on **My page** and wait for inbound message. | App remains on My page without navigation. |

### Phase B — Trigger message from ARCH

| Step | Action | Expected Result |
|------|--------|-----------------|
| 3 | Open **ARCH** for **SQA** or **UAT** (match app). Select **Skills** as **"[つか] Message (Soluto)"** or **"[故] Chat (KFS)"** per requirement (*this case: **KFS***). Sign in as **Operator**. | Correct environment + skill selected; Operator session is ready. |
| 4 | Send chat message from **ARCH** to target user. | Message is sent successfully from ARCH. |

### Phase C — Verify snack bar display behavior

| Step | Action | Expected Result |
|------|--------|-----------------|
| 5 | Observe app while user is on **My page** and message arrives. | Snack bar is displayed: **"新しいメッセージが届きました"**. |
| 6 | Navigate to **Timeline** page and receive/send another ARCH message for validation. | Snack bar is **not displayed** on Timeline page. |
| 7 | Navigate to **Diagnostics** page and receive/send another ARCH message for validation. | Snack bar is **not displayed** on Diagnostics page. |
| 8 | Navigate to **Search** page and receive/send another ARCH message for validation. | Snack bar is **not displayed** on Search page. |
| 9 | Navigate to **Contact Hub** page and receive/send another ARCH message for validation. | Snack bar is **not displayed** on Contact Hub page. |

---

## Expected result details

Message snack bar text:

```text
新しいメッセージが届きました
```

Display rule:

- Snack bar is displayed **only on My page**.
- Snack bar is **not displayed** on **Timeline**, **Diagnostics**, **Search**, **Contact Hub** pages.

---

## Expected result (overall)

When ARCH sends a **KFS** chat message, the message snack bar appears only on My page and does not appear on Timeline/Diagnostics/Search/Contact Hub pages.

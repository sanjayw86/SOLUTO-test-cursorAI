## Test Case 40: Verify `open_route_by_freq_asked` event value in Denodo report

**Test Case ID:** DB-DENODO-OPEN-ROUTE-FREQ-ASKED-001  
**Test Case Title:** Verify that `open_route_by_freq_asked` value is displayed in event log when user opens content from `م‚ˆمپڈمپ‚م‚‹مپٹه•ڈمپ„هگˆم‚ڈمپ›` list in `report_vfaq_analytics_report_realtime`  
**Priority:** High  
**Automation ID:** C00646  
**Grade:** C

**Preconditions:**
- App installed.
- Provisioning done.
- Tsukasapo user account available.
- Access to Denodo app/report `report_vfaq_analytics_report_realtime`.

**Test Steps:**

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Launch app. | App is launched. |
| 2 | Tap **Search** on footer. | Search screen opens. |
| 3 | Tap `م‚ˆمپڈمپ‚م‚‹مپٹه•ڈمپ„هگˆم‚ڈمپ›` to open list. | FAQ list opens. |
| 4 | Open any content from the list. | Content opens successfully. |
| 5 | Open Denodo app and search report `report_vfaq_analytics_report_realtime`. | Report is displayed successfully. |
| 6 | Validate column `metadata_eventname`. | `open_route_by_freq_asked` is displayed under `metadata_eventname`. |

**Test Data:**
- Brand/user: Tsukasapo provisioned user.
- Entry: `م‚ˆمپڈمپ‚م‚‹مپٹه•ڈمپ„هگˆم‚ڈمپ›` list.
- Report: `report_vfaq_analytics_report_realtime`.

**Denodo query (realtime):**
```sql
SELECT *
FROM enterprise_acyan.report_vfaq_analytics_report_realtime
where metadata_eventname like '%open_route_by_freq%'
order by collectiontime desc
limit 100;
```

**Denodo query (historic, next day):**
```sql
SELECT *
FROM enterprise_acyan.report_vfaq_analytics_report_historic
where metadata_eventname like '%open_route_by_freq%'
order by collectiontime desc
limit 100;
```

**Postconditions:**
- Evidence captured for app content open action and Denodo report row.

## Test Case 13: Call Reservation SMS is sent when reservation is made (Un-provisioned MDN, AU)

**Test Case ID:** SMS-CALL-RESERVATION-AU-001  
**Test Case Title:** Verify Call Reservation SMS is sent and recorded in DB when a reservation is made for an un-provisioned MDN (service enabled: true, brand AU)  
**Priority:** High  
**Automation ID:** C00557  
**Grade:** D  
**Preconditions:**
- Call reservation SMS conditions: **MDN un-provisioned**, **service enabled: true**, **brand: AU**.
- Un-provisioned MDN(s) are available (or can be enrolled via CSV upload or Orange API).
- Access to Soluto Timeslot Web CC tool and/or Shop UI tool; access to ISS DB for verification.

**Test Steps:**

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Have one or more un-provisioned MDNs. If needed, enroll MDN(s) via CSV upload or Orange API so they meet the conditions (un-provisioned for call reservation, service enabled: true, brand AU). | Un-provisioned MDN(s) are ready for call reservation. |
| 2 | Go to Soluto Timeslot Web **CC tool** or **Shop UI tool** and make a reservation using one of the test MDNs. | Reservation is created successfully. |
| 3 | In ISS DB, run: `select * from sms_system.message order by created_datetime desc limit 10;` Filter or identify the row where **Source = SUPPORT_SESSION** (Reservation category) for the test MDN. | Call Reservation SMS record is present in ISS DB for the test MDN. |
| 4 | Verify the message text for the Reservation SMS in the DB. | Text matches: **【使い方サポート】　お電話のご予約を承りました。ご予約情報は、使い方サポートアプリにてご確認いただけます。専用アプリはこちらからダウンロードしてください。 http://kddi-l.jp/eSd** (Call reservation SMS transmission is completed; confirmation is via log.) |

**Expected result (overall):** Call Reservation SMS transmission is completed and verifiable in ISS DB (Source = SUPPORT_SESSION, Reservation category).

**Test Data:**

- **MDN creation requirements (when creating an MDN):**
  - ① "使い方サポート" (usage support) contract  
  - ② Date of birth  
  - ③ Gender  
  - ④ Brand  

- **CC Tool:** https://callcenter.soluto-kddi-sqa.jpnmob-acyan.npr.aws.asurion.net/reservation/search  
- **Shop UI Tool:** https://shop.soluto-kddi-sqa.jp.nonprod-asurion53.com/auShop-entry  

- **Verification queries (備考):**
  - ISS DB (Reservation SMS – Source = SUPPORT_SESSION):  
    `select * from sms_system.message order by created_datetime desc limit 10;`  
  - In DB: **Source = SUPPORT_SESSION** indicates Reservation category (Reservation / Reminder / Cancellation SMS).

**Note:** Call reservation SMS is confirmed via DB/log; actual receipt cannot be verified. In DB, CARRIER ENROLLMENT = Enrollment SMS (Welcome+Reminder1+Reminder2); SUPPORT_SESSION = Reservation-related SMS.

**Postconditions:** Call Reservation SMS record exists in ISS DB for the test MDN. No mandatory cleanup unless required by project rules.

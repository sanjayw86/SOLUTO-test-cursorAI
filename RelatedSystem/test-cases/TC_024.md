## Test Case 24: ARCH Soluto chat — Contact Hub (Mypage), Operator, URL, closure

**Test Case ID:** ARCH-CHAT-SOLUTO-CONTACTHUB-001  
**Test Case Title:** Verify two-way chat, auto-reply, history after re-entry, adviser URL, and closure — **Operator** on **[つか] Message (Soluto)** when user opens consult from **Mypage → Contact Hub** (steps **10–11**), then steps **12–25** per phased tables below.  
**Priority:** High  
**Automation ID:** C00131  
**Grade:** D  

**Role:** **Operator** (must exchange chats; not Viewer / Business admin–only).

**Preconditions:**

- **User:** **au Tsukasapo** test user; **chat is not already open** for this session.  
- **Data:** IRIS (or dependent) data created using the **Stub** tool, per project procedure ([IRIS Stub Tool](https://uap-devtools-nonprod.jpnmob-acyan-npr.asurionpa.com/#/pascal/register) if applicable).  
- **Agent:** **Operator** account; **routing profile** set per **Routing profile** below.  
- **ARCH:** **Soluto** message context (**[つか] Message (Soluto)**), not **KFS**, for this test case (confirm **Contact Hub** lead lands on **Soluto** for your build).  
- **Agent status (Amazon Connect):** **受付可 (Available)** — required to receive chats (*per internal ARCH doc*).  
- **Browser:** **Chrome** for ARCH.  
- **Environment alignment:** Use **one** line end-to-end (e.g. **SQA1** app + **SQA1** ARCH + **SQA** Connect). Do not mix SQA app with UAT ARCH.

**Environments (URLs):**

| Layer | Purpose | URLs / notes |
|--------|---------|----------------|
| **ARCH** | Operator UI | **SQA1:** `https://acyan-sqa1.uap.jpnmob-acyan.npr.aws.asurion.net/techarch/` **SQA4:** `https://acyan-sqa4.uap.jpnmob-acyan.npr.aws.asurion.net/techarch/` **UAT1:** `https://acyan-uat.uap.jpnmob-acyan.npr.aws.asurion.net/techarch` **UAT2:** `https://acyan-uat2.uap.jpnmob-acyan.npr.aws.asurion.net/techarch` |
| **Amazon Connect** | Routing profile | **SQA:** `https://uap-acyan-sqa1-jpmob-telephony.my.connect.aws/home` **UAT:** `https://uap-acyan-uat-jpmob-telephony.my.connect.aws/home` |
| **App** | Tsukasapo | Build must match ARCH line; **UAT URLs** may change — confirm with **IT** if needed. |

**Routing profile (Connect):**

After Twilio → Connect, **routing profile** in Connect replaces skill selection in ARCH for queue routing. For **Soluto**, use e.g. **SOLUTOGeneralRoutingProfile** (confirm with team). **Wrong profile** may land chats on **[故] Chat (KFS)**. **App entry for this test:** **Mypage → ご相談・お手続き お問い合わせ → メッセージでご相談** → **Soluto**.

**How to set routing profile:** (1) Open **Connect** for SQA or UAT. (2) **User Management**. (3) Select **User**, search **your name**. (4) Open user → **Routing profile** dropdown → e.g. **SOLUTOGeneralRoutingProfile** → **Save** if required. (5) **ARCH** → **refresh** → verify **Routing Profile**. (6) If wrong, **set the profile again** in Connect (repeat from step 3), refresh ARCH, recheck.

**Test Steps:**

### Phase A — Connect routing profile + ARCH readiness

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Open **ARCH** for **SQA** or **UAT** (match app). Select **Skills** as **"[つか] Message (Soluto)"** or **"[故] Chat (KFS)"** per requirement (*this case: **Soluto***). Sign in as **Operator** when prompted. | Correct env + skill; **Operator** logged in (not Viewer-only). |
| 2 | On ARCH, view **Routing Profile** vs requirement for step 1 (e.g. **SOLUTOGeneralRoutingProfile** — confirm with team). | Profile visible and assessed. |
| 3 | Agree environment (e.g. SQA1). Open **Amazon Connect** (SQA or UAT, **Environments** table). | Connect home loads. |
| 4 | Open **User Management**. | User Management opens. |
| 5 | Select **User**, search **your name**. | User appears in results. |
| 6 | Select **your username**; set **Routing profile** (e.g. **SOLUTOGeneralRoutingProfile**). **Save** if required. | Profile assigned/saved. |
| 7 | Return to **ARCH**; **refresh**; confirm **Routing Profile**. If wrong after refresh, set profile again in Connect (repeat steps 5–6), refresh, recheck. | **Routing Profile** matches requirement. |
| 8 | Set status to **受付可 (Available)**. | Agent ready to receive chats. |
| 9 | Left menu → **Operation status** → check **available** agent counts. | Page loads; counts visible / recordable. |

### Phase B — App: open consult chat (Soluto path, Contact Hub entry)

Steps **10–11** use **Mypage → [ご相談・お手続き お問い合わせ] (Contact hub) → [メッセージでご相談]** (not **調べる → 困った・使い方がわからない**).

| Step | Action | Expected Result |
|------|--------|-----------------|
| 10 | On **Mypage**, tap **[ご相談・お手続き お問い合わせ]** (**Contact Hub**). | Screen opens as designed. |
| 11 | Tap **[メッセージでご相談]**. | Chat box opens. |
| 12 | Observe chat thread / message list. | **Previous / existing chat history** should be displayed. |
| 13 | Send some message from the **chat bot** (app chat / message input). | **Initial auto message** is sent (content depends on brand / business hours / availability — see Auto-Reply Messages in internal docs). |

### Phase C — History after back and re-entry

| Step | Action | Expected Result |
|------|--------|-----------------|
| 14 | Tap **Back**. | User leaves chat screen. |
| 15 | Tap **[メッセージでご相談]** again. | Chat box opens again. |
| 16 | Observe thread. | **Chat history** from steps **12–13** is **displayed**. |

### Phase D — Two-way messages + URL

| Step | Action | Expected Result |
|------|--------|-----------------|
| 17 | On **ARCH**, **accept** / open the session; confirm user context (e.g. MDN / app info). | Session active on adviser side. |
| 18 | User sends one or more messages. | Messages **arrive on ARCH** and show on the app. |
| 19 | **Adviser** sends some message(s) from **ARCH**. | Messages **should be received on** the **App**. |
| 20 | **Adviser** sends a message containing **au.com** (or full **https://au.com** per test plan). | Message delivered to app. |
| 21 | User opens the URL from the chat. | **URL opens** (browser or in-app behaviour per design). |

### Phase E — Close chat (ARCH + classification)

| Step | Action | Expected Result |
|------|--------|-----------------|
| 22 | At **ARCH** side, tap **[完了する]** (top right). | Wrap-up / finish flow appears. |
| 23 | Tap **[Finish]** (top right). | Closure fields appear (incl. red-highlighted areas per UI guide). |
| 24 | Select **問合せ区分**, **問合せ対象デバイス**, **終了理由** as required. | All mandatory fields set. |
| 25 | Tap **OK**. | Chat ends; app returns to the **Mypage** or **previous screen** if chat was open (*per ARCH behaviour*). |

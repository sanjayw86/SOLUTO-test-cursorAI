## Test Case 37: Contact hub chat — App/ARCH two-way message visibility (Soluto + KFS)

**Test Case ID:** ARCH-CHAT-BIDIRECTIONAL-VISIBILITY-BOTH-001  
**Test Case Title:** Verify app-to-ARCH and ARCH-to-app message visibility in Soluto and KFS chat routes  
**Priority:** High  
**Automation ID:** ARCH_TEMP_002  
**Grade:** D

**Role:** **User** (Tsukasapo/Kosapo app), **Agent/Operator** (ARCH).

**Preconditions:**

- **App** installed.  
- **Provisioning** completed.  
- **User:** Tsukasapo user (and KFS-capable account/build when running KFS route).  
- **ARCH:** Operator is signed in and can handle both contexts:
  - **[つか] Message (Soluto)** for Soluto run
  - **[故] Chat (KFS)** for KFS run
- **Browser:** Chrome for ARCH.  
- **Environment alignment:** One line end-to-end (app + ARCH + Connect if used); do not mix SQA app with UAT ARCH.

**Execution mode:** Run the same steps twice.

| Run | Chat type | ARCH skill/context | App entry summary |
|-----|-----------|--------------------|-------------------|
| **A** | **Soluto** | **[つか] Message (Soluto)** | Contact hub -> Chat (Soluto route) |
| **B** | **KFS** | **[故] Chat (KFS)** | Contact hub -> Chat (KFS route) |

**Environments (URLs):**

| Layer | Purpose | URLs / notes |
|--------|---------|----------------|
| **ARCH** | Operator UI | **SQA1:** `https://acyan-sqa1.uap.jpnmob-acyan.npr.aws.asurion.net/techarch/` **SQA3:** `https://acyan-sqa3.uap.jpnmob-acyan.npr.aws.asurion.net/techarch` **SQA4:** `https://acyan-sqa4.uap.jpnmob-acyan.npr.aws.asurion.net/techarch/` **SQA5:** `https://acyan-sqa5.uap.jpnmob-acyan.npr.aws.asurion.net/techarch/` **UAT1:** `https://acyan-uat.uap.jpnmob-acyan.npr.aws.asurion.net/techarch` **UAT2:** `https://acyan-uat2.uap.jpnmob-acyan.npr.aws.asurion.net/techarch` |
| **Amazon Connect** | Routing/agent (if used by line) | **SQA:** `https://uap-acyan-sqa1-jpmob-telephony.my.connect.aws/home` **UAT:** `https://uap-acyan-uat-jpmob-telephony.my.connect.aws/home` |
| **App** | Tsukasapo / Kosapo | Build aligned to selected ARCH line |

**Test Steps (execute for Run A and Run B):**

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Launch app. | App is launched. |
| 2 | Tap **Contact hub** on footer. | Contact page is opened. |
| 3 | Click **Chat** option. | Chat field/screen is opened. |
| 4 | Send a chat message from app. | Chat is received at ARCH (agent/operator can see it). |
| 5 | Send a chat message from ARCH. | Message is displayed at app side. |
| 6 | Send another chat message from app. | Chat is received at ARCH again. |

---

## Expected result (overall)

For both Soluto and KFS runs, message visibility is successful in both directions:

- App -> ARCH messages are visible to agent/operator.
- ARCH -> App messages are visible in app chat.
- Follow-up App -> ARCH message is also visible.

## Test Case 28: ARCH chat push notification reception

**Test Case ID:** ARCH-CHAT-SOLUTO-PUSH-NOTIFICATION-001  
**Test Case Title:** Verify push notification is received when ARCH sends message to au Tsukasapo user while app is closed  
**Priority:** High  
**Automation ID:** C00077  
**Grade:** D

**Role:** **Operator** (send-side from ARCH).

**Preconditions:**

- **User:** **au Tsukasapo** test user.
- **App state:** App is **not launched** (closed/background-killed as per project policy).
- **Notification settings:**
  - **OFF:** Soluto app in-app notification switch.
  - **ON:** OS push notification for the app.
- **ARCH context:** Soluto chat flow (Tsukasapo) available for the target user.
- **Skill selection (ARCH):** Select **[つか] Message (Soluto)** for this test case. Use **[故] Chat (KFS)** only if project scope explicitly requires KFS validation.

- **Environment alignment:** Use one line end-to-end (app + ARCH). Do not mix SQA app with UAT ARCH.

## Expected result (overall)

When ARCH sends chat message to an au Tsukasapo user and app is not launched, the user receives push notification on OS (with app notification OFF and OS push ON).

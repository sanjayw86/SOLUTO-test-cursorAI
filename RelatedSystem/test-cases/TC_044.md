# Manual Test Cases: CC Tool — Call reservation and cancel (Tsukasapo)

**Source:** CC Reservation Tool, Shop Tool, Tsukasapo app (Mypage / push)  
**Viewpoint:** Reserve/cancel call flows with **Reserve/Cancel — ON & ON** (Soluto app notification **ON**, OS push notification **ON**)  
**Purpose:** Verify reservation and cancellation behaviour when actions originate from CC Tool, mobile, or Shop Tool, including push titles and Mypage reservation visibility.

**Sync rule (apply in the steps below):** Whenever a reservation is **booked** from **any** application or device, it must appear **on both sides** — on the **Tsukasapo device (Mypage / in-app)** **and** in/on **the same application or tool where it was booked** (for example CC Reservation Tool, Shop Tool). Whenever it is **cancelled** from **any** side, it must be **cancelled on both sides** — **device** and **that particular application/tool** stay aligned.

## Test Case 44: CC Tool / Shop Tool / mobile — call reserve and cancel (sync both sides)

**Test Case ID:** CC-CALL-RESERVE-CANCEL-MULTI-001  
**Test Case Title:** Verify call reservation and cancellation across CC Tool, mobile, and Shop Tool with expected push and Mypage state  
**Priority:** High  
**Automation ID:** C00092  
**Grade:** D

**Role:** Tester (CC Tool / Shop Tool access + Tsukasapo test device).

**Preconditions:**

- **User:** Tsukasapo user; **provisioning completed**.  
- **App:** **Not launched** at start of step 1 (per scenario: reserve from CC while app closed).  
- **Settings:** **Soluto app notification: ON**; **OS push notification: ON**.  
- **Feature flags / mode:** **Reserve/Cancel call — ON & ON** (per tracker).  
- **Environment:** Selected **per requirement** (SQA, UAT, etc.); tools and app build aligned to that line.  
- URLs for CC Reservation Tool and Shop Tool recorded from [UAT_60. Access Information](https://asurion.atlassian.net/wiki/spaces/ANSFK/pages/123336319/UAT_60.+Access+Information).

**Test Steps:**

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | **Reserve call** on **CC Reservation Tool** while the **mobile app is closed** (not launched). | **Push notification** received with **Title:** `【お電話のご予約を承りました】`. Reservation appears **on both sides:** **CC Reservation Tool** shows the booking **and**, after opening the app, **device (Mypage)** shows the **same** reservation (**sync rule**). |
| 2 | On the device, **confirm reservation result** (open app / Mypage as per team flow). **Also** check **CC Reservation Tool** for the same MDN/session. | **Reservation information is displayed** on **Mypage** **and** the **same** reservation is **visible in CC Reservation Tool** — **both sides** match. |
| 3 | **Cancel** the reservation **on your device** (mobile). **Then** verify **CC Reservation Tool**. | **Reservation information is not displayed** on the **device** (Mypage / in-app cleared) **and** the reservation is **cancelled / not shown** in **CC Reservation Tool** — **both sides** (**sync rule**). |
| 4 | **Reserve call** from the **mobile** side. **Then** verify **CC Reservation Tool** (or team-paired admin view) if applicable. | Reservation is **shown on both sides:** **device (Mypage)** **and** **the application/tool** that reflects mobile bookings (for example **CC Reservation Tool** per environment) — **same** booking, **both sides** (**sync rule**). |
| 5 | **Cancel** the call reservation on **CC Tool**. **Then** verify the **device**. | **Push notification** received with **Title:** `【お電話のご予約がキャンセルされました】`. **Reservation information is not displayed** on the **device** **and** the booking is **cancelled / not shown** in **CC Tool** — **both sides** (**sync rule**). |
| 6 | **Reserve call** again from the **mobile** side. **Then** verify the paired tool/application. | Reservation is **active on both sides:** **device (Mypage)** **and** **CC Reservation Tool** (or applicable tool) show the **same** reservation before the next cancel (**sync rule**). |
| 7 | **Cancel** the call reservation on **Shop Tool**. **Then** verify the **device**. | **Reservation information is not displayed** on the **device** **and** **Shop Tool** shows the reservation **cancelled / cleared** — **both sides** (**sync rule**). |

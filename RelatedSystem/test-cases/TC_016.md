## Test Case 16: Call Reservation SMS is not sent when reservation is made (Provisioned MDN, AU)

**Test Case ID:** SMS-CALL-RESERVATION-NOSEND-AU-001  
**Test Case Title:** Verify Call Reservation SMS is not sent when a reservation is made for a provisioned MDN (service enabled: true, brand AU)  
**Priority:** High  
**Automation ID:** C00664  
**Grade:** D  
**Preconditions:**
- Call reservation SMS conditions: **MDN provisioned**, **service enabled: true**, **brand: AU**. When MDN is provisioned, Call Reservation SMS will not be sent.
- Provisioned MDN(s) are available (e.g. provisioned via Orange API or project provisioning process).
- Access to Soluto Timeslot Web CC tool and/or Shop UI tool; access to ISS DB for verification.

**Test Steps:**

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Have one or more **provisioned** MDNs (e.g. provision via Orange API or the project's provisioning process so that the MDN is provisioned, service enabled: true, brand AU). | Provisioned MDN(s) are ready for call reservation. |
| 2 | Go to Soluto Timeslot Web **CC tool** or **Shop UI tool** and make a reservation using one of the test MDNs. | Reservation is created successfully. |
| 3 | In ISS DB, run: `select * from sms_system.message order by created_datetime desc limit 10;` Check for records where **Source = SUPPORT_SESSION** (Reservation category) for the test MDN. | **No** Call Reservation SMS record is present in ISS DB for the test MDN (Call Reservation SMS is not sent for provisioned MDN). |

**Expected result (overall):** Call Reservation SMS will not be sent when MDN is provisioned (verifiable via DB/log).

**Test Data:**

  - ④ Brand

- **Verification queries (備考):**

    `select * from sms_system.message order by created_datetime desc limit 10;`  
  - In DB: **Source = SUPPORT_SESSION** indicates Reservation category; for provisioned MDN, Call Reservation SMS should not be sent.

**Note:** Call reservation SMS confirmation is via DB/log; actual receipt cannot be verified. For provisioned MDN, Call Reservation SMS is not sent.

**Postconditions:** No Call Reservation SMS record exists in ISS DB for the test MDN (as expected for provisioned MDN). No mandatory cleanup unless required by project rules.

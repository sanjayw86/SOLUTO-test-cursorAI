## Test Case 4: Welcome SMS sent by next day when MDN meets OWL/BISCUIT conditions (UQ)

**Test Case ID:** SMS-WELCOME-004  
**Test Case Title:** Verify Welcome SMS and Reminder 1/2 SMS are scheduled and sent by next day for new MDN (Orange API, brand UQ)  
**Priority:** High  
**Preconditions:**
- OWL/BISCUIT conditions are met: **service enabled: true**, **brand: UQ**.
- Orange API registration is enabled for the environment.
- Postman (or equivalent) is available; access to Titan DB and ISS DB for verification queries.

**Test Steps:**

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | In Postman, create a new account/MDN that meets the OWL/BISCUIT conditions in advance (see Test Data for MDN requirements). | MDN/account is created with ① "使い方サポート" contract, ② Date of birth, ③ gender, ④ brand. |
| 2 | Send the POST request to the carrier enrollment API. In the request body, change only **MDN** and **SubscriberCode** as needed for the test. | Request is accepted (2xx); enrollment is registered in the system. |
| 3 | In Titan DB, run: `select * from carrier_integration.message_target mt order by created_datetime desc;` | One or more rows appear for the test MDN; SMS is scheduled (message_target records created). |
| 4 | In ISS DB, run: `select * from sms_system.message order by created_datetime desc limit 100;` | Recent records include the test MDN; messages are present for the enrollment. |
| 5 | In ISS DB (and/or Titan views as per 備考), verify that Welcome SMS and Reminder1 and Reminder2 SMS are scheduled to be sent by the next day for that MDN. | Welcome SMS and Reminder 1 & 2 SMS are scheduled and sent by the next day for the test MDN. |

**Expected result (overall):** Welcome SMS will be sent by the next day.

**Test Data:**

- **MDN creation requirements (when creating an MDN):**
  - ① "使い方サポート" (usage support) contract  
  - ② Date of birth  
  - ③ Gender  
  - ④ Brand  

- **POST request (curl example):**
```bash
curl --location --request POST 'https://carrier.soluto-kddi-uat.jpnmob-acyan.npr.aws.asurion.net/api/carrier/enrollment' \
--header 'Content-Type: application/json' \
--header 'X-Api-Key: 611fb6c3-882d-4da8-9172-849de6523c8c' \
--data-raw '{
  "source": "01",
  "mdn": "<change as needed>",
  "subscriberCode": "<change as needed>",
  "serviceCode": "K211",
  "startDate": "20220411",
  "subscriptionType": "01"
}'
```

- **Verification queries (備考):**
  - Titan view (realtime):  
    `SELECT * FROM titan_views.sms_system_message_realtime_view ssmrv WHERE user_id = 'c8da98db-6a11-4375-aa9e-3884104822d1' ORDER BY created_datetime DESC LIMIT 10;`
  - Titan view (mat view):  
    `SELECT * FROM titan_views.sms_system_message_mat_view ORDER BY created_datetime DESC LIMIT 10;`
  - Titan:  
    `SELECT * FROM carrier_integration.message_target mt ORDER BY created_datetime DESC;`
  - ISS:  
    `SELECT * FROM sms_system.message ORDER BY created_datetime DESC LIMIT 100;`

**Postconditions:** Test MDN has Welcome and Reminder 1/2 SMS scheduled/sent; DB rows can be used for traceability. No mandatory cleanup unless test data must be removed per project rules.
